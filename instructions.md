# Proyek Akhir Pemrograman Berorientasi Objek Lanjut (Java)
## Smart Manufacturing Work Order Management System

### Latar Belakang
Sebuah perusahaan manufaktur memiliki banyak mesin produksi dan operator yang bekerja secara bersamaan. Saat ini pencatatan work order masih dilakukan secara manual sehingga sering terjadi duplikasi pekerjaan, konflik penggunaan mesin, kesalahan input data, dan sulit memonitor status produksi secara real-time.

### Tujuan Pembelajaran
1. Melakukan analisis dan desain sistem menggunakan OOAD.
2. Mengimplementasikan OOP secara benar.
3. Menggunakan Exception Handling untuk menangani kesalahan sistem.
4. Mengimplementasikan Multithreading untuk simulasi proses produksi paralel.
5. Menghasilkan aplikasi yang modular dan mudah dikembangkan.

### Deskripsi Sistem
Sistem mengelola data Mesin, Operator, dan Work Order.

#### Bagian A - Object Oriented Analysis and Design (OOAD)
Mahasiswa wajib membuat Use Case Diagram, Class Diagram, Sequence Diagram, dan Activity Diagram.

#### Bagian B - Implementasi OOP
Implementasikan class Machine, Operator, WorkOrder, Production Manager, dan ProductionThread.

#### Bagian C - Exception Handling
Implementasikan custom exception: `MachineUnavailableException`, `InvalidQuantityException`, dan `DuplicateWorkOrderException`.

#### Bagian D - Multithreading
Setiap Work Order harus dijalankan menggunakan thread terpisah untuk simulasi produksi paralel.

#### Bagian E - Sinkronisasi Thread
Gunakan `synchronized` untuk mencegah konflik penggunaan mesin oleh beberapa thread.
